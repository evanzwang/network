                                            README
                                        Landing Zones
                                      This repository contains different types of Landing Zones to deploy to the Oracle Cloud Infrastructure platform. The landing zones are assembled from modules and templates that users can use in their default configuration or fork this repo and customize for your own scenarios.
                                        Oracle Enterprise Landing Zone v2 (OELZ v2)
                                      The Oracle Enterprise Landing Zone v2 (OELZ v2) deploys a typical architecture used by enterprise customers. The root template for this landing zone is located at
                                        . Users can use the guides below to get started with the Oracle Enterprise Landing Zone v2 (OELZ v2).
                                          [Architecture Guide](/niya-ma-1/oci-landing-zones/blob/main/Official_Documentation/OELZ_Baseline_Deployment/Architecture_Guide.md)
                                          [Implementation Guide](/niya-ma-1/oci-landing-zones/blob/main/Official_Documentation/OELZ_Baseline_Deployment/IMPLEMENTATION.md)
                                          [Configuration Guide](/niya-ma-1/oci-landing-zones/blob/main/Official_Documentation/OELZ_Baseline_Deployment/CONFIGURATION.md)
                                        Workload Expansion
                                      The Oracle Enterprise Landing Zone v2 (OELZ v2) deploys a workload in each environment (Prod and Non-Prod) by default.
The user can use the workload expansion stack to deploy additional customized workload. The template for the workload expansion is located
                                      at
                                        [Official_Documentation/OELZ_Workload_Deployment](/niya-ma-1/oci-landing-zones/blob/main/Official_Documentation/OELZ_Workload_Deployment)
                                        . Users can use the guides below to get started with Workload Expansion.
                                        Deploy Using Oracle Resource Manager
                                        Click to deploy the stack
                                          If you aren't already signed in, when prompted, enter the tenancy and user credentials. Review and accept the terms and conditions.
                                        Select the region where you want to deploy the stack.
                                        For Working directory, select
                                          templates/enterprise-landing-zone
                                        Follow the on-screen prompts and instructions to create the stack.
                                        After creating the stack, click Terraform Actions, and select Plan.
                                        Wait for the job to be completed, and review the plan.
                                        To make any changes, return to the Stack Details page, click Edit Stack, and make the required changes. Then, run the Plan action again.
                                        If no further changes are necessary, return to the Stack Details page, click Terraform Actions, and select Apply.
